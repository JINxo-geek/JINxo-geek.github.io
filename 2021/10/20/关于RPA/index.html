Robotic process automation (RPA)

在未来，那些你不想做的枯燥工作，也许真的可以不做。重复化、标准化的工作都可以让RPA机器人帮你完成。

RPA通过模拟人工手动操作键鼠，自动处理规则清晰、批量化的高频业务。它适用于企业内具有明确业务规则、结构化输入和输出的操作流程，例如读取邮件、对账汇总、检查文件、生成文件和报告等枯燥、重复、标准化的工作，都可以让RPA机器人代为完成。

RPA能够轻松集成在任何系统上，跨系统处理数据。不仅简化了操作流程，提高了数据处理效率和准确度，还可有效避免人为失误，规避业务流程中数据被人为篡改的风险。通过RPA可以减轻员工的工作量，将员工从枯燥、重复的数据搬运中释放出来，让他们专注于更具创造性的工作，成长为复合型人才，更好地利用数据，为企业创造更多价值。

眼下，RPA已被广泛应用于金融、保险、零售、电商、政府、能源、制造、物流、地产、医疗、教育、电信等行业，在财务管理、人力资源、客服、法务、采购等职能场景也发挥着重要作用。

随着RPA技术的不断发展，其与AI（Artificial Intelligence，人工智能）的结合也已成为行业发展的一大趋势。引入AI能力的RPA机器人将在文字识别、文本理解和人机对话等方面获得重大突破，从而为用户带来更大程度的自动化。

推荐使用的

1. UiPath (https://www.uipath.com/） (中文网址：https://www.uipath.com.cn/）UiPath是我在与不同咨询公司以及同行交流中知名度最高的。

优点（Pros）：

  • 有无限期的个人免费版本，和60天企业免费试用版本。
  • 免费的线上课程，互动性强。有视频，讲义，练习题和答案，学完每一课有考试。不仅针对想成为RPA developer的人群有设计对应的课程，如果想做RPA项目经理，RPA分析学完之后拿到证书。
  • 软件代码语法源于http://vb.NET，有基础的学起来会比较快。
  • 完整的生态系统，有自己的论坛供大家提问交流，有其他人或者团队已经开发好的代码分享。平台还提供完整的数据表提供KPI和一些关键的metrics。支持用户提交任务给RPA组评估其自动化的可行性。

缺点（Cons）：

  • 价格对传统行业的中小型企业来说还是比较贵的。之所以这么说是因为比较难找到应用的案例来达到ROI的平衡。 不光是licensing的费用，还需要配置RPA developer来管理这些bots。对于传统行业来说，能自动化的流程肯定存在，但是既达到重复性，稳定性（自动化对象的软件与流程不常变化），规则简单（可以画出完整流程图包括特例）又累计耗费人工成本大于RPA成本的流程，就比较少了。大多数情况下，核心业务会耗费大量人工成本，但其对应的流程可能会复杂多变，我会建议寻找专门为企业定制开发RPA的公司而不是由自己的公司来投入资本进行开发。

2. Automation Anywhere (AA）

和UiPath是劲敌。两者是在RPA领域里占市场份额最大的商业化公司。

优点（Pros）：

  • 提供无限期的个人免费试用版本。
  • 对于小型企业来说，基础配置的价格比UiPath的同类型要友好很多。
  • 提供软件学习的课程。

缺点（Cons）：

  • 目前了解到的是AA从搭建RPA到使用到实际中的周期要比UiPath长。UiPath的平台可以做到六个月之内，而AA却可能需要更长的时间来完成。从市场来看，AA的年收入增长没有UiPath快，知名度没有UiPath大。我近期也会做一个详细针对AA的调查来找出为什么市场更喜爱UiPath的原因。

3. TagUI

优点（Pros）：

  • 免费的开源工具。这里附上下载链接：

TagUI Command Line Github: https://github.com/kelaberetiv/TagUI

TagUI Python Github: https://github.com/tebelorg/RPA-Python

  • 适用于Windows，macOS 和 Linux。支持网页，鼠标/键盘，OCR，图像识别的自动化。详细的关于TagUI介绍请点用户指南。
  • 可通过Python API来使用TagUI
  • 可以通过API/Python/R来与不同的深度学习模型结合！聊过的Data Scientist当中就有人在进行这方面的实操。

缺点（Cons）：

  • 对于代码基础薄弱/不熟悉CLI的用户可能就不太友好了。
  • 和收费的RPA工具相比一些对应的配置不齐全。我个人觉得TagUI适应于一些规则简单却需要许多bots同时运行的流程。毕竟收费的RPA工具大多数是按bot来收费的而TagUI最大的优点就是其免费性。

4. Olive

这家公司是专门针对医疗行业提供RPA服务。其商业模式是给在医疗行业中的企业或医院安装他们的bot然后我们通过发放固定工资的形式支付这些bots所提供的服务。所有的维护费管理费均包括在了“工资”里。也就是现在逐渐流行起来的AI-as-a-Service(AIaaS) model.

5.按键精灵和Uibot

按键精灵和UIbot是同一家公司的团队出品的,目前按键精灵处于停止维护,要体验最新功能还是需要使用Uibot比较好.
